# v53 — Managed calendar event preferences + dry-run sync (no-op schema change)

- No schema-level changes in this release.
- `CurrentSchemaVersion` bumped from 52 to 53 to track the release line.
- Behavioral change: introduces a managed-calendar preferences popout, a
  duplicate-safe sync reconciliation flow, a dry-run confirmation step
  before mutating managed calendars, and alignment of iCal preferences
  with the v52 Chicago timezone semantics.
- Manual steps: none. Existing archives migrate transparently because the
  schema is unchanged.
- See `git log v1.2.52..v1.2.53` for the commit range.
