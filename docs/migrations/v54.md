# v54 — Calendar sync UX hardening (no-op schema change)

- No schema-level changes in this release.
- `CurrentSchemaVersion` bumped from 53 to 54 to track the release line.
- Behavioral change: hardens the calendar sync UX and the popout layout
  for managed and test Google calendars. Improves error surfacing when
  sync conflicts arise.
- Manual steps: none. Existing archives migrate transparently because the
  schema is unchanged.
- See `git log v1.2.53..v1.2.54` for the commit range.

> Note: a v55 schema bump that added image-compression metadata
> (`compressed_at`, `original_bytes`, `compressed_bytes` on the `images`
> table) was introduced in commit `264df7f` and reverted in `ea99f07`
> after real-world benchmarking showed JPEG q=85 re-encode grew 71% of
> files. The revert left the schema at v54. The v55 migration note
> shipped briefly with `264df7f` and was removed with the revert; no
> v55 archive exists in the wild.
