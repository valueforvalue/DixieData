# v58 — Person Record tagging (issue #183)

- Add `tags`, `person_record_tags`, and `archive_meta` tables.
- `tags` carries a `name` column (display-cased) and a
  `normalized_name` UNIQUE column for case-insensitive uniqueness.
- `person_record_tags` is the many-to-many join between `tags`
  and `soldiers` (Person Records). ON DELETE CASCADE on both
  sides so a tag delete drops every membership and a Person Record
  delete drops every tag binding.
- `archive_meta` carries per-archive-kind export toggles. Seeded
  with three rows: `shared_archive` (tags OFF), `backup_archive`
  (tags ON — full SQLite snapshot), `static_archive` (tags OFF,
  HTML export excludes working notes).
- New bilingual indexes for the join table so the Browse filter
  (`GROUP BY person_id HAVING COUNT(DISTINCT tag_id) = N`) and
  per-soldier membership reads stay sub-millisecond.

Existing archives migrate transparently. `tags` and
`person_record_tags` start empty so no backfill is needed; the
seed `archive_meta` rows are inserted on legacy upgrades via the
`ensureArchiveMetaSeed` helper called at the tail of `applySchema`.

Bumps `CurrentSchemaVersion` to 58. No manual steps.

Rollback: `bump-version.ps1` without `-Force` refuses any
downgrade that would drop a table or index required by live code.
