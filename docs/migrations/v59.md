# v59 — saved Share Queue presets (issue #192)

- Add `share_queue_presets` table.
- `name` UNIQUE column carries the user-facing label
  (e.g. "Shiloh Cemetery", "4th Alabama Roster").
- `soldier_ids_json` carries a JSON array of integer
  soldier IDs (the v1 schema keeps the payload simple --
  no separate join table -- because preset membership is
  small and we never query "all presets containing soldier X"
  in reverse).
- `created_at` + `last_used_at` columns mirror the
  `export_templates` shape from v58 so the two surfaces
  feel identical to power users.
- Local-only storage: no `sync_id` column. Saved presets
  are personal working-set memory, not shared state.

Existing archives migrate transparently. The table starts
empty so no backfill is needed; the `CREATE TABLE IF NOT
EXISTS` block in the schema constant runs unconditionally
during applySchema, and the `PRAGMA user_version = 59`
update fires at the end.

Bumps `CurrentSchemaVersion` to 59. No manual steps.

Rollback: `bump-version.ps1` without `-Force` refuses any
downgrade that would drop a table or index required by live
code.
